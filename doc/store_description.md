# OutBlocker - 快適なブラウジング体験を取り戻す

OutBlockerは、「戻る」ボタンを押した際に表示される迷惑な広告（ブラウザバック広告）を自動的に検出してブロックする拡張機能です。History APIの不正利用を監視・防止し、ユーザーが意図した通りにウェブブラウジングを楽しめるようサポートします。

## 主な機能

### 🛡️ 完全ブロック機能
- **History API監視**: 広告挿入のためのHistory API操作を検出・ブロック
- **リソースブロック**: 広告関連ドメインからのリソース読み込みを防止
- **DOM要素削除**: ページ内の広告要素を自動的に検出・削除

### ⚙️ カスタマイズ設定
- **シンプルな操作**: ワンクリックで拡張機能の有効/無効を切り替え
- **サイト別設定**: 特定のサイトでのみ有効/無効にする設定が可能
- **統計情報**: ブロックした広告数を確認できる機能

### 🔒 プライバシー重視
- **データ収集なし**: ユーザーデータを一切収集・送信しません
- **軽量設計**: 最小限のリソースで動作し、ブラウジング体験に影響を与えません
- **オープンソース**: コードは透明性を確保するため公開されています

## こんな方におすすめ

✓ ブラウザバック時に表示される広告にストレスを感じている方
✓ ウェブ閲覧時のナビゲーションをスムーズに保ちたい方
✓ 予期しない広告ページへのリダイレクトを防ぎたい方
✓ プライバシーを重視し、追跡を最小限に抑えたい方

## 技術的な仕組み

OutBlockerは以下の技術を使用して広告をブロックします：

1. **History API監視**: 
   ブラウザのHistory API（history.pushStateとhistory.replaceState）の不正利用を検出し、広告挿入のための操作をブロックします。

2. **リソースブロック**:
   広告配信に関連するドメインからのリクエストを検出し、ブロックします。これにより広告コンテンツの読み込みを防止します。

3. **DOM要素の監視と削除**:
   MutationObserverを使用してDOM変更を監視し、広告関連の要素が追加された場合に即座に削除します。

## プライバシーポリシー

OutBlockerは、ユーザーのプライバシーを最大限に尊重するよう設計されています：

- ユーザーデータを外部サーバーに送信しません
- 広告ブロック統計はローカルに保存されます
- 閲覧履歴や個人情報へのアクセスは行いません
- 必要最小限の権限のみを使用します

## 使用方法

1. 拡張機能をインストールすると、自動的に有効化されます
2. ブラウザの右上に表示される拡張機能アイコンをクリックすると、設定画面が開きます
3. 設定画面では以下の操作が可能です:
   - 拡張機能全体の有効/無効の切り替え
   - 現在閲覧中のサイトでの拡張機能の有効/無効の切り替え
   - ブロックした広告数の確認
   - 統計情報のリセット

## 必要な権限について

OutBlockerは以下の権限を使用します：

- `webRequest`: 広告関連リソースの読み込みをブロックするため
- `history`: ブラウザ履歴の不正操作を検出するため
- `tabs`: 現在のタブ情報にアクセスし、サイト別設定を適用するため
- `storage`: 設定と統計情報を保存するため

これらの権限は広告ブロック機能を実現するために必要最小限のものです。
